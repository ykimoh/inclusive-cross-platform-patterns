Pattern 06 · Annotate

Focus order & keyboard navigation annotation

Annotate the design with the focus path, focus-trap regions, and skip-link placement so engineers implement keyboard navigation correctly the first time.

Problem

Mouse users can click anywhere; keyboard users tab through DOM order. When DOM order doesn’t match the visual reading order, keyboard navigation lands users in unexpected places, the cart icon before the product list, the footer before the “continue” button. Custom widgets without keyboard handlers are unreachable. Modals without focus traps and Esc-to-close strand users mid-flow.

Engineers can’t infer the intended focus order from a visual mockup alone, especially for complex layouts where DOM order may need to differ from visual order to support both keyboard and mouse use.

Pattern

  • Numbered focus path. Annotate each interactive element on the design with a sequence number matching the intended tab order. The numbers are the spec; engineers wire DOM and tabindex to match.
  • Skip-to-content link. First focusable element on every page. Visible on focus only; lands focus on #main.
  • Focus-trap regions. Modal dialogs, custom comboboxes, drawer overlays. Mark the trap boundary, name the close action, and specify Esc-to-close.
  • Focus indicators. Every interactive element has a visible focus indicator with at least 3:1 contrast against the page surface and 2px minimum width. Never outline: none without a replacement.
  • Roving tabindex. For composite widgets (toolbars, listboxes, radio groups), one tab stop per widget; arrow keys move within. Annotate which key moves where.

Visual examples

Visual examples (a design with the numbered focus path overlaid, plus a modal dialog with focus-trap boundary marked) will be added during pattern review.

Code

The annotated focus path translates to plain DOM order plus :focus-visible styling. Avoid positive tabindex values; use semantic source order.

<!-- Skip link: first focusable, visible on focus only -->
<a class="skip-link" href="#main">Skip to main content</a>

<!-- Main is a focus target after skip -->
<main id="main" tabindex="-1">
  ...
</main>
/* Focus indicator: 2px solid, 3:1 contrast against the surface */
:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 3px;
  border-radius: 1px;
}

/* Skip link: off-screen until focused */
.skip-link {
  position: absolute;
  top: -3rem;
  left: 1rem;
  padding: 0.5rem 1rem;
  background: var(--accent);
  color: #fff;
  z-index: 100;
}
.skip-link:focus { top: 1rem; }

Annotation

On the design, place a numbered circle next to each interactive element (1, 2, 3 …). For focus-trap regions, draw a boundary box with the trap label and close-action note. Document non-default key bindings (Esc, arrow keys for composite widgets).

Focus path: 1 → 2 → 3 → 4 (skip link, primary nav, search, main CTA)
Focus trap: Modal "Add to cart"
Trap close: Esc, click backdrop, X button (all return focus to invoking button)
