Inclusive Cross-Platform Patterns

A field-facing reference for designers who practice inclusive cross-platform UX. Twelve patterns across the practitioner journey from empathy to validation.

About

This library documents twelve patterns for designing inclusive interfaces that work across web, iOS, and Android. Each pattern is a single page with the problem, the pattern, visual examples, a copy-pasteable code snippet, the WCAG 2.1 success criteria it addresses, and a short reading list.

Most pattern libraries organize by component type. This one organizes by what the designer is doing in the moment, surfacing barriers, applying audits and inclusive content choices, annotating the design for handoff, validating with assistive technology before engineering picks it up.

1 · Empathize

  • 01.Disability-scenario empathy primer

    Practitioner-facing scenario cards that walk through a single task from a user-with-disability perspective, surfacing exactly where the typical UI breaks down.

2 · Design

  • 02.Color contrast audit + accessible alternatives

    Identify failing color pairs against WCAG 2.1 AA (4.5:1 body text, 3:1 large text and non-text), and resolve them within an existing design-system palette.

  • 03.Inclusive UX copy

    Identify exclusive verbs and assumptions in interface copy and provide an inclusive-substitution glossary, e.g. “watch this video” → “play this video.”

  • 04.Inclusive imagery

    Audit image content for representation across age, gender, ability, race, and culture; provide a checklist for sourcing photography and illustration.

  • 05.Inclusive identity forms

    Form patterns for gender identity, pronoun, and assigned-sex-at-birth fields that support open-ended self-identification rather than binary defaults.

  • 11.Adaptive onboarding state

    Onboarding flows that progressively disclose features based on detected state (first-run, screen-reader-detected, prefers-reduced-motion) rather than a fixed wizard.

3 · Annotate

  • 06.Focus order & keyboard navigation annotation

    How to annotate keyboard tab order, focus-trap regions, and skip-link placement on a design file so engineers implement them correctly.

  • 07.Structure landmarks annotation

    How to annotate ARIA landmark regions so screen-reader users can skip-jump through page structure rather than reading top-to-bottom.

  • 08.Alt text annotation

    When to write alt text vs. mark an image decorative; how to write functional alt text for product imagery, icons, and decorative illustration.

4 · Validate

  • 09.Accessibility-mode validation walkthrough

    A repeatable, designer-runnable pre-handoff check: walk the design with keyboard + VoiceOver/NVDA to surface focus traps, missing labels, and broken landmark order.

  • 10.Accessibility testing with people with disabilities

    How to design and run an unmoderated accessibility test session with users of assistive technology: recruitment criteria, task framing, moderation considerations, analysis.

Cross-cutting

  • 12.Cross-platform consistency

    How a single pattern (focus indicator, form label, landmark structure) maps consistently across web, iOS, and Android, with notes on where the platforms diverge by necessity.
